# Style Defaults (tex-repo)

This document describes the **default presentation style** generated by tex-repo
for book-scale and paper-scale documents.

Style defaults are provided as a practical starting point.
Authors may freely modify presentation after scaffolding.
tex-repo never validates or enforces visual style.
All automation (`status`, `guard`, `fix`) operates on structure only.

---

## 1. Shared style foundation

Both book and paper documents share a common baseline under:

```text
shared/
  packages.tex   - LaTeX package imports
  preamble.tex   - Central include entry
  macros.tex     - Custom command definitions
  notation.tex   - Mathematical notation shortcuts
  identity.tex   - Document metadata (title, author, etc.)
```

The `shared/` directory avoids duplication and provides a consistent
starting environment across documents.

### 1.1 Design goals

The shared baseline is:

* **Readable**
  Suitable for long-form technical writing

* **Conservative**
  Uses widely available LaTeX packages

* **pdflatex-compatible**
  No dependency on XeLaTeX or LuaLaTeX

* **Minimal**
  No theming, colors, or decorative elements

### 1.2 Baseline package set

At initialization, tex-repo generates `shared/packages.tex`
with a stable set of commonly used packages, grouped by purpose.

**Fonts and encoding**

```tex
\usepackage[T1]{fontenc}
\usepackage{lmodern}
\usepackage{microtype}
```

**Page layout**

```tex
\usepackage[margin=1in]{geometry}
```

**Mathematics**

```tex
\usepackage{amsmath, amssymb, amsthm}
\usepackage{mathtools}
```

**Graphics and tables**

```tex
\usepackage{graphicx}
\usepackage{booktabs}
```

**Links and cross-references**

```tex
\usepackage[hidelinks]{hyperref}
\usepackage[nameinlink,noabbrev]{cleveref}
```

The baseline is intentionally minimal and expected to remain stable.

### 1.3 Shared preamble behavior

Every document entry file includes:

```tex
\input{shared/preamble.tex}
```

The preamble includes shared files in a fixed order:

1. `packages.tex`
2. `macros.tex`
3. `notation.tex`
4. `identity.tex`

Document-class-specific configuration remains in the entry file,
not in `shared/`.

## 2. Book style (book-class documents)

### 2.1 Document class and role

Long-form documents use the `book` class to support:

* Part and chapter hierarchy
* Native numbering behavior
* Frontmatter, mainmatter, and backmatter separation

### 2.2 Book entry template (default)

```tex
\documentclass[12pt]{book}

\input{shared/preamble.tex}

\title{Introduction}

\begin{document}

\frontmatter
\maketitle
\tableofcontents
\input{build/sections_index.tex}

\mainmatter
\input{build/chapters_index.tex}

\backmatter
% appendices, references, indices

\end{document}
```

### 2.3 Typography defaults

Book templates are generated with the following defaults:

* No paragraph indentation
* Visible paragraph spacing
* Stable vertical layout (`\raggedbottom`)
* Conservative heading spacing via `titlesec`
* Latin Modern font with `microtype`

Defaults are applied only at scaffolding time.

### 2.4 Customization

After creation, authors may modify:

* Fonts and layout via `shared/packages.tex`
* Spacing and sectioning in the entry file
* Macros and environments via `shared/macros.tex`

tex-repo does not reapply or enforce book style.

---

## 3. Paper style (article-class documents)

### 3.1 Document class

Paper-scale documents use the `article` class.
They are intended to be:

* Standalone
* Compact
* Manually structured by the author

### 3.2 Paper entry template (default)

```tex
\documentclass{article}

\input{shared/preamble.tex}

\title{Paper Title}
\author{Author Name}

\begin{document}

\maketitle

\section{Introduction}
% content

\end{document}
```

Sectioning is authored manually.

### 3.3 Typography defaults

Paper style shares the shared baseline:

* Latin Modern font
* `microtype` enabled
* 1-inch margins
* Minimal ornamentation

### 3.4 Customization

Authors may:

* Add document-specific packages in the entry file
* Override geometry or spacing locally
* Apply journal-specific styles

Paper style is fully author-controlled after creation.

---

## 4. Relationship to automation

Style defaults are applied **only at scaffolding time**:

* `init`, `book`, and `paper` generate initial templates
* `build` regenerates structural spines only
* `fix` restores missing structure, not style
* `status` and `guard` never inspect presentation

Structural correctness and visual presentation are intentionally decoupled.

---

## 5. Summary

* Shared style provides a conservative, readable baseline
* Book style supports long-form hierarchical writing
* Paper style supports compact standalone documents
* Authors retain full control after scaffolding

**Principle:** tex-repo enforces structure; authors own presentation.

---

## 6. Internal template references

tex-repo maintains internal reference templates:

```
texrepo/templates/
  shared_packages.tex
  shared_preamble.tex
```

These files are **internal to the tex-repo tool** and serve as
scaffolding sources.

Generated repositories do **not** contain these files.
End users edit only the generated files:

```
shared/
  packages.tex
  preamble.tex
  macros.tex
  notation.tex
  identity.tex
```

Do not modify internal template files unless contributing to tex-repo itself.
